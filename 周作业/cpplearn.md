## 第二天/3.18
构造函数调用规则
- 默认构造函数，默认析构函数（无 参，函数体为空）默认拷贝构造函数（对属性值拷贝）
- 若用户定义有参构造函数，cpp不再提供默认无参构造，但会提供默认拷贝构造
- 若用户定义拷贝构造函数，cpp不再提供其他函数

深拷贝与浅拷贝
- 如果在类里面建立一个属性指针，并为他new一个空间，若此时该类存在多个对象（即默认拷贝构造函数创建对象），则会出现多个指针控制同一内存的情况;当我们释放某一指针内存时，会出现其他指针指向一被释放空间的情况。
- 解决方式：自己定义一个拷贝构造函数来实现创建对象的属性拷贝并另开一个空间来控制（自主定义构造函数是需要养成的好习惯）

- 所有对象共享同一段数据
- 编译阶段分配内存
- 类内声明，类外初始化（int 类::变量=数值；）
- 注意：1.该变量/函数不属于任何对象，可通过对象（p.m_name）来访问或者通过类(Person::m_name)来访问
- 2.存在访问权限（private或portect）
- 3.静态成员函数可以访问静态成员变量不可以访问非静态成员变量4.成员变量和成员函数分开储存
  
this指针（指针常量）
- cpp编译器会给每个空对象分配一个字节空间，是为了区分空对象占内存的位置
- 非静态成员变量属于对象上，静态成员变量/函数/非静态成员函数不属于对象上
- 概念：指向被调用的成员函数所属对象（每个非静态成员函数只会诞生一份函数实例，也就是说多个同类型对象会共用同一块代码）
- 用途：1.解决名称冲突
- class Person{\
      - int age\
      - Person(int age){\
            - this->age=age\
      - }
- }
- this->age指向的是该对象的age属性，将构造函数的参数值赋给该对象age属性
- 2.返回对象本身用*this
- 当函数返回类型不同（如void）可用return *this;(this是指向该对象的指针，*this直接指向该对象本体)

## 第三天/3.19
const修饰成员函数
- const：只读
- 为了防止变量被意外改动，通常用const修饰。
- this指针本质上是Person * const this(指针指向确定后不可更改)
- 若想让指针指向值也不可修改，则const Person * const this,转变为函数形式就是 void 函数名（）const,即修饰的是this指针，且指向变量值也不可修改
- 特殊变量：mutable//即使在常函数中，也可以修改这个值，只需要在类型前加上关键字mutable

常对象
- 在创建对象前加上const变为常对象，属性不可修改（mutable属性除外）
- 常对象只能凋用常函数（因为普通成员函数可以修改属性）

友元（关键字friend）
- 三种实现：
- 1.全局函数做友元
- 2.类做友元
- 3.成员函数做友元
- 
- 1.全局函数前加上关键字friend即可访问private属性
- 2.类（2）做友元即在类1中声明（friend class 类2），类2可以访问类1中的private成员
- 3.成员函数（2）做友元即在类1中声明 friend 函数类型 类2：：函数名（）；，意思是类2成员函数作为类1友元，可访问类1private成员
